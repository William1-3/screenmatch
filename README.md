# Screenmatch

Aplicação desenvolvida em Java com Spring Boot para consulta, gerenciamento e persistência de informações sobre séries e filmes.

O projeto foi desenvolvido durante os estudos de Java e Spring, aplicando conceitos de desenvolvimento backend, integração com APIs externas e persistência de dados utilizando Hibernate.

## Tecnologias Utilizadas

- Java
- Spring Boot
- Spring Web
- Spring Data JPA
- Hibernate
- Maven
- PostgreSQL
- API Externa de Filmes e Séries
- Jackson (JSON)

## Funcionalidades

- Busca de séries por meio de API externa
- Conversão de respostas JSON para objetos Java
- Persistência de dados em banco de dados
- Consulta de séries cadastradas
- Filtragem de informações
- Organização em camadas seguindo boas práticas de desenvolvimento

## Conceitos Aplicados

### Spring Boot

Utilização do Spring Boot para configuração e inicialização da aplicação, reduzindo código de infraestrutura e facilitando a criação de aplicações Java modernas.

### Spring Web

Implementação da camada responsável pela comunicação da aplicação e processamento das requisições.

### Hibernate e JPA

Mapeamento objeto-relacional (ORM) para persistência dos dados, permitindo a manipulação de entidades Java sem a necessidade de escrever consultas SQL complexas manualmente.

### Consumo de APIs

Integração com serviços externos para obtenção de informações sobre séries e filmes, utilizando requisições HTTP e tratamento de respostas em formato JSON.

### Maven

Gerenciamento de dependências, build e organização do projeto.

## Estrutura do Projeto

O projeto segue uma arquitetura em camadas, separando responsabilidades entre:

- Model
- Repository
- Service
- Configurações da aplicação
- Camada de acesso a dados

## Aprendizados

Durante o desenvolvimento deste projeto foram praticados conceitos fundamentais para aplicações backend modernas, incluindo:

- Programação Orientada a Objetos
- Consumo de APIs REST
- Persistência de dados
- Spring Framework
- Hibernate
- Manipulação de JSON
- Records
- Streams
- Banco de Dados Relacional
